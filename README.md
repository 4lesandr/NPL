# NPL2 - Коллекция программ на различных языках программирования

Этот проект содержит 15 программ, разделенных на 5 категорий, демонстрирующих различные парадигмы и возможности языков программирования.

## Структура проекта

### 1. Многопоточность (3 программы)
- **Rust** - параллельная обработка данных из файла с использованием потоков
- **Erlang** - параллельная обработка данных из файла с использованием процессов (actor model)
- **Scala** - параллельная обработка данных из файла с использованием Future и акторов

### 2. Рекурсивные алгоритмы (3 программы)
- **Haskell** - вычисление чисел Фибоначчи с мемоизацией
- **Lisp** - вычисление чисел Фибоначчи с мемоизацией
- **OCaml** - вычисление чисел Фибоначчи с мемоизацией

### 3. Конвертация данных (3 программы)
- **Ruby** - преобразование CSV в JSON
- **Julia** - преобразование CSV в JSON
- **Groovy** - преобразование CSV в JSON

### 4. Клиент-серверное взаимодействие (3 программы)
- **D** - TCP сервер и клиент
- **Elixir** - TCP сервер и клиент (GenServer)
- **Swift** - TCP сервер и клиент

### 5. Парсинг данных (3 программы)
- **Lua** - извлечение email, телефонов и IP-адресов из текста с помощью регулярных выражений
- **ML** - извлечение email, телефонов и IP-адресов из текста
- **Objective-C** - извлечение email, телефонов и IP-адресов из текста с помощью NSRegularExpression

## Документация - структура входных данных

### 1. Многопоточность
Входные данные представляют собой текстовый файл (`data.txt`), содержащий строки с данными для обработки. Каждая строка может содержать числа, текст или их комбинацию. Пример:
```
12345
67890
abc123def456
test789
```

### 2. Рекурсивные алгоритмы
Входные данные не требуются - программы вычисляют числа Фибоначчи для заданного индекса, который обычно передается как параметр командной строки или задается в коде.

### 3. Конвертация данных
Входные данные представляют собой CSV файл (`data.csv`) с заголовками в первой строке и данными в последующих строках. Пример:
```csv
name,age,city,email
Иван,25,Москва,ivan@example.com
Мария,30,Санкт-Петербург,maria@example.com
```

### 4. Клиент-серверное взаимодействие
Входные данные не требуются - программы реализуют TCP сервер и клиент, которые обмениваются сообщениями через сетевые сокеты.

### 5. Парсинг данных
Входные данные представляют собой текстовый файл (`input.txt`), содержащий произвольный текст с встроенными email-адресами, номерами телефонов и IP-адресами. Пример:
```
Email: ivan.petrov@example.com
Телефон: +7 (495) 123-45-67
IP адрес сервера: 192.168.1.100
```
